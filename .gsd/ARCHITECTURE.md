## Data Flow

1. User enters a command in the `Input` widget.
2. `on_input_submitted` intercepts the command text.
3. If the command falls into a restricted set of builtins (`cd`, `clear`, `history`, `sysinfo`, `exit`, `help`), it is handled internally.
4. Otherwise, it delegates to `subprocess.run` running via the underlying shell.
5. `stdout` and `stderr` are collected and appended to the `RichLog` widget, along with a custom styled prompt.
